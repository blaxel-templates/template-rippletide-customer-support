Overview
Rippletide provides an SDK to build reliable and governed agents. Blaxel provides a secure sandbox platform that lets you host agents next to them for near instant performances.

The Rippletide Ã— Blaxel integration lets you:
- Create an agent using Rippletide SDK
- Configure your agent with Q&A pairs, tool calls, guardrails, and user inputs
- Evaluate your agent by extracting questions from a PDF and testing responses

1. Prerequisites
Before starting, ensure you have:
- A Rippletide API key (get it from https://eval.rippletide.com)
- A Blaxel account
- Python 3.10+
- uv package manager (recommended)
- Git installed

Optional:
- Basic JSON knowledge
- Understanding of REST APIs

2. Clone the Integration Template
The official template is available here:
https://github.com/blaxel-ai/template-rippletide-customer-support

Clone the repository:
git clone https://github.com/blaxel-ai/template-rippletide-customer-support 
cd template-rippletide-customer-support

Install dependencies:
uv sync

This installs all required libraries for Rippletide + Blaxel.

3. Configure Your API Key
Get your Rippletide API key:
1. Go to https://eval.rippletide.com
2. Login and go to settings
3. Generate your API key

Update the hardcoded API key in these files:
- src/setup_agent.py - Update RIPPLETIDE_API_KEY = ""
- src/agent.py - Update RIPPLETIDE_API_KEY = "your-api-key-here"

4. Understanding Agent Configuration
Your agent's logic lives inside the agent_config.json file:
- agent_purpose: The main prompt describing your agent's role
- qa_pairs: Pairs of questions and ground-truth answers
- guardrails: Rules and constraints you want to apply to your agent
- tool_calls: External API calls your agent can make
- user_input_collection: Information your agent needs to collect from users
- format_answer: Instructions for formatting responses
- state_predicate: Rules for when the agent should end conversations

ðŸ‘‰ To customize your agent, simply edit agent_config.json.

5. Create and Evaluate Your Rippletide Agent
Copy the example configuration:
cp agent_config.json.example agent_config.json

Edit agent_config.json with your configuration (optional).

Run the setup script with a PDF containing questions:
uv run src/setup_agent.py agent_config.json --pdf knowledge.pdf

This script performs the following:
- Loads your agent configuration
- Creates a brand-new SDK agent in your Rippletide workspace
- Configures Q&A pairs, tool calls, guardrails, and user inputs
- Creates an evaluation agent
- Extracts questions and expected answers from the PDF
- Asks the SDK agent the first question from the PDF
- Gets the agent's answer
- Evaluates the answer against the expected answer from the PDF
- Prints the evaluation report
- Returns an Agent ID

6. Blaxel Platform Setup
Complete Blaxel setup by following the quickstart guide:
https://docs.blaxel.ai/Get-started#quickstart

Install Blaxel CLI:
curl -fsSL https://raw.githubusercontent.com/blaxel-ai/toolkit/main/install.sh | BINDIR=/usr/local/bin sudo -E sh

Login to Blaxel:
bl login

7. Deploy the Agent on Blaxel
Blaxel handles hosting and runtime execution for your Rippletide agent.

a. Test locally
Run the local development sandbox:
bl serve --hotreload

This launches a local environment exposing your agent's inference endpoint.

Test the agent:
bl chat --local template-rippletide-customer-support

b. Deploy to production
bl deploy

Blaxel will build and deploy your project, then generate a public inference endpoint, for example:
https://run.blaxel.ai/{YOUR-WORKSPACE}/agents/{YOUR-AGENT}

This URL is now your production-ready agent endpoint.

You can call it from:
- Customer support tools
- Websites
- Back-office apps
- CRMs
- Slack/Teams bots
- Internal systems

8. Calling Your Agent
Send a POST request to your Blaxel endpoint:
curl -X POST https://run.blaxel.ai/YOUR-WORKSPACE/agents/YOUR-AGENT \
  -H "Content-Type: application/json" \
  -H 'X-Blaxel-Authorization: Bearer YOUR-TOKEN' \
  -H 'X-Blaxel-Workspace: YOUR-WORKSPACE' \
  -d '{
    "message": "Hi, I need help with my recent order.",
    "user_id": "customer_123"
  }'

The request is processed as follows:
- Input â†’ forwarded to your LLM
- LLM generates hypotheses
- Hypotheses â†’ validated, corrected, and authorized by Rippletide
- Rippletide selects the only safe, allowed action
- Action is executed by your runtime
- Response is returned to the caller

9. Evaluating Your Agent
The evaluation is integrated into the setup script. When you run:
uv run src/setup_agent.py agent_config.json --pdf knowledge.pdf

The script automatically:
- Creates an SDK agent using your configuration
- Creates an evaluation agent
- Extracts questions and expected answers from the PDF
- Asks the first question to the SDK agent
- Evaluates the answer against the expected answer
- Prints the evaluation report

The PDF parameter is required - it provides the questions and expected answers for evaluation.

10. Troubleshooting
The agent fails to create
- Check your API key in src/setup_agent.py
- Ensure RIPPLETIDE_API_KEY is set correctly
- Make sure you have a PDF file with questions for evaluation

Blaxel deploy fails
Run:
bl auth login

The agent behavior is incorrect
Re-run the setup after modifying agent_config.json:
uv run src/setup_agent.py agent_config.json --pdf knowledge.pdf

No questions extracted from PDF
- Ensure your PDF contains extractable Q&A pairs
- Check that the PDF file path is correct
- The script will try to get test prompts from the evaluation agent if PDF extraction returns no results

Endpoint returns an error
Try bl serve locally first to debug.
